Master’s Thesis — Hella GmbH & Co. KGaA (FORVIA)

Optimization of Modal Parameters for Headlamp Components Under Random Vibration

Oct 2024 – Aug 2025 · Erlangen, Germany · Master’s thesis & internship

Note: a paper based on this work is being prepared.

Goal in simple words

A headlamp experiences random vibration in a vehicle. I built a process to make a computer model behave like the real part by tuning/optimizing a few physical parameters (material/connector stiffness and damping). When the model and the test match well, engineers can trust the model to make quicker design decisions.

What I did in the lab

  • Set up random-vibration tests on a shaker and measured motion with a 3D camera and accelerometers.
  • Prepared multiple build states (bare housing, with modules, and with controlled connector play) and ensured clean boundary conditions and sensor placement.
  • Processed raw data into targets we can compare to simulation: overall motion (RMS displacement), representative mode shapes, frequency-response function, and PSD curves using Modal View software and Python scripts.
  • Aligned coordinates & markers so that the test points and model points refer to the same locations and directions using Zeiss GOM software.

How I built the simulation

  • Pre-processed the assembly in HyperMesh: created the mesh, defined boundary conditions/fixtures, and then added connector definitions to represent screws and snap-fits (with appropriate stiffness and damping).
  • Exported Abaqus input files containing the model mesh, material properties, connector definitions, and analysis steps: Frequency (eigenvalue), Steady-State Dynamics, and Random Response.
  • Kept the number of tunable parameters small so the results remain physical, explainable, and easy to maintain.
  • Checked the model’s basic behaviour (natural frequencies and mode shapes) before moving on to calibration.

How I matched model to experiment

  • Modal correlation: first aligned the FE model with experimental modal analysis by matching natural frequencies and mode shapes to establish a valid baseline.
  • Objective definition: minimize the error between the FEA and experiment in terms of displacement RMS (Urms) and PSD/FRF curves.
  • Automated calibration with HEEDS: used HEEDS (SHERPA) to drive a multi-objective, gradient-free search over modal parameters such as Rayleigh damping and connector stiffness/damping; Abaqus runs were scripted in Python, and HEEDS minimized the above error metrics to tune parameters until convergence.

Checking for non-linear behaviour

I also checked whether the connector play (small gaps/clearances) causes behaviour that a simple linear model cannot capture. Coherence plots (a measure of how consistent the signal is with the input) helped spot regions where contact or friction effects start to matter. For small gaps, the linear model remained predictive; larger gaps showed clear signs of non-linearity.

Transferability & outcomes

  • Applied the calibrated parameters first on one headlamp, then checked transferability on a second headlamp with a different structure; the method held up well.
  • Clear guidance on when the quick linear model is sufficient and when a more detailed contact model is worth it.

Correlation summary: after calibration, the remaining differences between experiment and simulation were typically within about 10% in the vertical (Z) direction and within about 25% overall.
